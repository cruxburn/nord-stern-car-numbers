📋 FIELD DESCRIPTIONS
─────────────────────

🔑 PRIMARY KEY
• id: Auto-incrementing unique identifier

🔒 UNIQUE CONSTRAINT  
• car_number: Ensures no duplicate car numbers

⚠️  REQUIRED FIELDS
• first_name: Driver's first name
• last_name: Driver's last name  
• car_number: Car number (unique)

📝 OPTIONAL FIELDS
• car_make: Manufacturer (BMW, Porsche, Audi, etc.)
• car_model: Model name (M3, 911, RS4, etc.)
• car_year: Manufacturing year
• car_color: Vehicle color
• reserved_date: Date when number was reserved
• notes: Additional information

⚙️  AUTO-MANAGED FIELDS
• reserved_for_year: Defaults to 2025
• status: Defaults to 'Active'
• created_at: Set automatically on creation
• updated_at: Updated automatically on changes

🎯 BUSINESS RULES
─────────────────

📊 Car Number Management:
• Stored as TEXT with leading zeros (e.g., "001", "014", "123")
• Automatically formatted to 3 digits with leading zeros
• Must be unique across all registrations

📈 Status Values:
• Active: Currently registered and participating
• Retired: No longer active but record preserved  
• Pending: Registration in progress

⏰ Timestamp Management:
• created_at: Set once when record is created
• updated_at: Updated every time record is modified
